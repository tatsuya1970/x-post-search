# 統合テスト仕様書

## 概要

X Post Search の統合テストは、複数コンポーネント・モジュールが連携して動作することを検証する。

## テスト環境

- **実行環境**: Electron + Next.js 静的エクスポート
- **外部依存**: X API (api.twitter.com)

## 統合テストシナリオ

### IT-01: 検索フロー（投稿検索）

| ステップ | 操作 | 期待結果 |
|----------|------|----------|
| 1 | アプリ起動 | メイン画面が表示される |
| 2 | Bearer Token を設定 | 設定が保存され、「API接続済み」と表示 |
| 3 | アカウント名入力・検索実行 | 件数確認 → 確認ダイアログ → 検索実行 |
| 4 | 結果表示 | テーブルにツイートが表示される |
| 5 | CSVエクスポート | 保存ダイアログが開き、ファイルが保存される |

### IT-02: 日付フィルタ連携

| ステップ | 操作 | 期待結果 |
|----------|------|----------|
| 1 | 抽出開始日・終了日を設定 | 日付が JST 基準で API に渡される |
| 2 | 今日を終了日に設定 | エラーにならず、現在時刻まで取得 |
| 3 | 過去の日付範囲で検索 | 該当期間のツイートのみ取得 |

### IT-03: いいね検索フロー

| ステップ | 操作 | 期待結果 |
|----------|------|----------|
| 1 | OAuth 4項目を設定 | ユーザー認証が有効になる |
| 2 | 自分のアカウントでいいね検索 | 検索が実行され、いいねしたツイートが取得される |
| 3 | 他人のアカウントでいいね検索 | アラートで「自分のアカウントのみ」と拒否される |

### IT-04: エラーハンドリング

| ステップ | 操作 | 期待結果 |
|----------|------|----------|
| 1 | 無効な Bearer Token で検索 | エラートーストが表示される |
| 2 | 存在しないユーザー名で検索 | 「ユーザーが見つかりませんでした」と表示 |
| 3 | Bearer のみでいいね検索 | 「ユーザー認証が必要」と表示 |

## 実行方法（手動）

1. `npm run build:mac:unsigned` でビルド
2. `dist/mac-arm64/X Post Search.app` を起動
3. 上記シナリオを手動で実行し、結果を記録
